# PopApp UI [Native StyleSheet. Native gestures. Native haptics. No Tailwind required.]

PopApp UI is a **shadcn-style component registry** for React Native and Expo. Copy beautiful, production-ready components into your project with a single command.

## Why PopApp UI?

:::success[Own Your Code]
PopApp follows the copy-and-own model — every component lives in your project, fully customizable. No opaque `node_modules`.
:::

- **StyleSheet-native** — no NativeWind, no Tailwind, no className resolution
- **Gesture-first** — spring physics and UI-thread animations via Reanimated
- **Haptic feedback** — built into every interactive component
- **Liquid Glass** — iOS 26+ glass effects where supported
- **TypeScript-first** — full type safety with exported interfaces
- **Expo-optimized** — works out of the box with Expo, compatible with bare RN

## Quick Start

::::steps

##### Initialize your project

```bash [Terminal]
npx popapp init
```

This scaffolds the directory structure and installs the theme system.

##### Add components

```bash [Terminal]
npx popapp add button card text-input
```

Dependencies are resolved automatically — adding `button` also installs `theme`, `haptics`, `glass`, and `touchable`.

##### Use in your app

```tsx
import { Button } from "@/components/ui/button";
import { Card } from "@/components/ui/card";

export function MyScreen() {
  return (
    <Card>
      <Button title="Get Started" onPress={() => {}} />
    </Card>
  );
}
```

::::

## Animation Tiers

Every component declares an animation tier so you know exactly what native dependencies are required.

| Tier | Dependencies | Description |
|------|-------------|-------------|
| **Tier 1** | None | Pure React Native. Zero animation dependencies. |
| **Tier 2** | `react-native-reanimated` | Spring animations on the UI thread. Included in Expo by default. |
| **Tier 3** | `reanimated` + `gesture-handler` | Full gesture control with pan, pinch, and swipe support. |
